Что строим в Спеке 1 — «Фундамент платформы»

Один спек = публикуем контракт агента + работающий портал-MVP, проверенный сквозь один echo-агент end-to-end. Дальше уже переносим под этот контракт твои реальные агенты — отдельным спеком, отдельной командой.

👤 Пользователь / Администратор — браузер

Преподаватель кафедры: выбор агента → загрузка файлов → форма параметров → ждёт прогресс → скачивает Word. Админ: управление вкладками, агентами, пользователями.

↓ HTTPS

🌐 Frontend

  • Главная: 3 вкладки, плитки агентов
  • Страница агента: форма + загрузка файлов/папок
  • Страница задачи: live-прогресс, логи
  • Админка: вкладки, агенты, юзеры

⚙️ Backend API

  • Auth (email + пароль, JWT)
  • Реестр агентов и вкладок
  • CRUD задач (создать / посмотреть / отменить)
  • WebSocket для live-логов и прогресса
  • Раздача артефактов на скачивание
↓

🗄️ Postgres

  • users, tabs, agents, agent_versions
  • jobs, job_events, artifacts
  • audit log, usage / лимиты

📋 Очередь задач

  • Job попадает в очередь → воркер берёт
  • Воркер стримит события прогресса в БД и WebSocket
  • Конкурентность настраиваемая

📦 Агент-Runner (Docker)

  • Один контейнер = одна задача (изоляция)
  • stdin: JSON c параметрами
  • stdout: NDJSON событий
  • $IN_DIR / $OUT_DIR — bind-mount
  • Лимиты по CPU / RAM / времени
↓

🤖 OpenRouter — LLM-gateway

Один ключ портала → агент дёргает Claude / GPT / Gemini / DeepSeek через единый API. Портал ставит лимиты, считает расход, логирует. Студенту не нужен свой ключ.

📁 Хранилище файлов

MVP — локальная FS на сервере портала. Позже — S3-совместимое (для бэкапа и роста). Загрузки пользователей + артефакты-результаты.
