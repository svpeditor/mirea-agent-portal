Манифест агента — на примере proverka

Файл manifest.yaml лежит в корне репозитория агента. Портал читает его, чтобы построить форму, навесить лимиты, изолировать запуск.

# --- ИДЕНТИФИКАЦИЯ --- id: school-works-expert name: "Эксперт конкурсных работ школьников" version: "1.0.0" category: "научная-работа" icon: "🎓" short_description: | Анализирует работы школьников по чек-листу и выдаёт экспертное заключение в Word.

Идентификация и место в портале

обязательно id — уникальный slug. По нему версионирование и URL.

обязательно category — одно из значений «научная-работа», «учебная», «организационная». Определяет вкладку.

опционально icon — emoji или путь к PNG. По умолчанию первая буква названия в кружке.

обязательно short_description — то, что видно на плитке. about ниже — длинная страница «о агенте».

# --- ФОРМА ВВОДА --- inputs: section: type: text label: "Секция конкурса" placeholder: "например: робототехника" default: "общая" required: true strict_mode: type: checkbox label: "Жёсткий режим" help: "Снижает балл за каждый недостаток" default: false

Параметры формы

Из этого блока портал автоматически рисует форму в UI. Студент описывает поле — пользователь его видит.

Поддерживаемые типы: text, textarea, number, checkbox, select, radio, date.

Для select/radio — поле options со списком значений.

Валидация: min, max, pattern, min_length, max_length.

# --- ФАЙЛЫ НА ВХОДЕ --- files: works_folder: type: folder label: "Папка с работами" help: "В каждой подпапке — текст работы и презентация" accept: [".docx", ".pdf", ".pptx"] max_total_size_mb: 500 required: true

Файлы и папки

Типы загрузки: single_file, multi_files, folder (с сохранением структуры подпапок), zip.

В UI это превращается в drag-n-drop зону. Для folder используется HTML5 directory upload + fallback на ZIP.

Внутри контейнера агент получает путь через $INPUT_works_folder. Структура исходных папок сохраняется.

# --- АРТЕФАКТЫ НА ВЫХОДЕ --- outputs: - id: report type: docx label: "Экспертное заключение" filename: "экспертное_заключение.docx" primary: true

Что вернёт агент

Список ожидаемых файлов. Портал проверяет что они появились — иначе job помечается как failed.

Типы: docx, pdf, xlsx, zip, html, json, any.

primary: true — что предлагается скачать первым большой кнопкой. Остальные — список рядом.

# --- КАК ЗАПУСКАТЬ --- runtime: docker: base_image: python:3.12-slim setup: - "pip install -r requirements.txt" entrypoint: ["python", "agent.py"] llm: provider: openrouter models: ["deepseek/deepseek-chat"] limits: max_runtime_minutes: 30 max_memory_mb: 1024 max_cpu_cores: 2

Runtime и лимиты

Docker: базовый образ + setup-шаги. Образ собирается один раз при добавлении агента и кэшируется.

LLM: агент декларирует, какие модели использует. Портал инжектит ключ OpenRouter в env. Студент в коде делает обычный OpenAI-клиент с base_url=$OPENROUTER_URL.

Лимиты: жёсткие потолки. Если агент висит дольше — kill. Защита от бесконечного цикла или утечки.
