## 1. Цель и контекст

После 1.2.3 агент успешно запускается в Docker-сэндбоксе с `network_mode=none` и обменивается с порталом через файлы + JSONL-stdout. Цель 1.2.4 — дать агенту **доступ к LLM (OpenRouter)** через прокси портала с **enforced квотами** на пользователя и cap'ом на одну задачу, чтобы:

- Агенты могли использовать LLM в работе (`proverka`, `science_agent`, etc. — без LLM они нерабочие).
- Один кривой агент не сжёг весь бюджет проекта за выходные.
- Каждый LLM-вызов был залогирован (модель, токены, стоимость, юзер, агент, job) — нужно для биллинга, аудита, и для расширения в 1.2.5 (per-agent квоты).
- Агент **физически не мог** выйти в интернет помимо нашего прокси (sandbox от 1.2.3 не должен ослабляться).

После merge 1.2.4 контракт `runtime.llm` из `docs/contract.md` (определён ещё в 1.1) становится рабочим: агент пишет `runtime.llm.provider: openrouter` + whitelist моделей, портал генерит ephemeral OpenRouter-ключ на каждый job и делает прокси с проверкой квот.

**Что становится возможным после merge:**
- Перенос реального `proverka` под контракт v0.1 (План 1.4) — он использует LLM.
- Запуск любого студенческого агента с openai-совместимым SDK (deepseek через `openai.Client(base_url=$OPENROUTER_BASE_URL)`).
- Frontend (1.3) при отображении job'а видит `cost_usd_total` и истёкшую квоту.

## 2. Scope

### Входит ✅

- Таблицы `user_quotas`, `llm_ephemeral_tokens`, `llm_usage_logs` + расширение `jobs.cost_usd_total` через Alembic миграцию `0004_llm_proxy`.
- Backfill `user_quotas` для существующих юзеров значениями по умолчанию.
- API роутер `/llm/v1/*` — OpenAI-совместимый прокси:
  - `POST /llm/v1/chat/completions` (включая `stream: true` SSE).
  - `POST /llm/v1/completions` (legacy, на всякий случай).
  - `GET /llm/v1/models` — отдаёт whitelist моделей текущего агента.
  - Прочие OpenAI-эндпоинты — `501 Not Implemented`.
  - Auth: `Authorization: Bearer <ephemeral_token>` (отдельный middleware `EphemeralTokenAuth`).
- Расширения существующих эндпоинтов:
  - `GET /api/me` + `GET /api/admin/users/{id}` — поле `quota: {monthly_limit_usd, period_used_usd, period_starts_at, per_job_cap_usd}`.
  - `GET /api/me/usage?from=&to=` — cursor-paginated история LLM-вызовов юзера.
  - `PATCH /api/admin/users/{id}/quota` — менять `monthly_limit_usd` / `per_job_cap_usd` индивидуально.
  - `POST /api/admin/users/{id}/quota/reset` — принудительный reset.
  - `GET /api/admin/usage` — aggregation для админа (top-users, top-models, total-cost).
- Изменение `POST /api/agents/{slug}/jobs`:
  - Если `agent_version.manifest.runtime.llm` присутствует — pre-flight quota check, генерация ephemeral_token, передача worker'у.
  - Если LLM не используется — поведение прежнее.
- Расширение `manifest_loader.py` (worker):
  - Валидация `runtime.llm.provider == "openrouter"`, non-empty `models`, каждая модель — в глобальном whitelist'е (env `LLM_ALLOWED_MODELS`).
- Изменение `docker_runner.py` (worker):
  - Два режима сети: `network_mode="none"` (без LLM) или `network="portal-agents-net"` (с LLM).
  - В LLM-режиме env `OPENROUTER_API_KEY` + `OPENROUTER_BASE_URL` пробрасываются в контейнер.
- Изменение `tasks/run_job.py` (worker):
  - При терминальном статусе job (succeeded/failed/cancelled/timed_out) — UPDATE `llm_ephemeral_tokens.revoked_at = now()` через свой sessionmaker.
- Сервисы portal-api:
  - `services/llm_pricing.py` — кеш модельных цен из OpenRouter с фоновым refresh.
  - `services/llm_quota.py` — pre-flight check, post-flight write, lazy month reset, FOR UPDATE locking.
  - `services/llm_proxy.py` — pass-through + streaming, валидация модели против manifest whitelist'а.
  - `services/ephemeral_token.py` — генерация / hash / resolve / revoke.
- `docker-compose.yml`:
  - Сеть `portal-agents-net` с `internal: true` (без NAT).
  - `portal-api` подключён к двум сетям (`portal-net` + `portal-agents-net`).
  - `portal-worker` остаётся в `portal-net` (Docker SDK сам присоединит agent-контейнер к agents-net).

### Не входит ❌

- Per-agent квоты (отложены до 1.2.5 — `llm_usage_logs` собирает данные с дня 1, чтобы решение было on-data).
- Embeddings, images, audio, function-calling специфичные для разных провайдеров эндпоинты (501).
- Многопровайдерность (Anthropic API, OpenAI напрямую). Только OpenRouter.
- Frontend для админ-квотных эндпоинтов (это часть 1.3).
- Cron для очистки протухших ephemeral-токенов (отложено, не критично).
- Webhook-уведомления юзеру о приближении к квоте (отложено).
- Soft-cap / warnings — только hard 402.

## 3. Архитектура

### Компоненты

- **`portal-api`** (расширяется) — добавляются роутер `/llm/v1/*`, middleware `EphemeralTokenAuth`, сервисы `llm_*`. Прокси не выделяется в отдельный сервис: на MVP логика тонкая (auth → check quota → forward → log usage), смешение с обычным API оправдано простотой деплоя.
- **`portal-agents-net`** — internal Docker network (без NAT в интернет). `portal-api` подключён к двум сетям одновременно: `portal-net` (Postgres + Redis + outbound к OpenRouter) и `portal-agents-net` (агентские контейнеры). Агенты резолвят `portal-api:8000/llm/v1/...` по embedded DNS Docker'а.
- **`portal-worker`** (расширяется) — новый параметр `LlmRuntimeConfig` в `run_agent_container`. Сам worker не входит в `portal-agents-net` (он лишь создаёт контейнеры через Docker SDK).

### Поток job с LLM

```
1. POST /api/agents/{slug}/jobs (auth: cookie JWT)
   → services/jobs.create_job:
     - check user_quota.period_used_usd < monthly_limit_usd (lazy reset перед чеком)
     - если manifest.runtime.llm → генерим ephemeral_token (uuid4),
       INSERT INTO llm_ephemeral_tokens (token_hash=sha256(plaintext), ...)
     - INSERT INTO jobs (...) → enqueue в RQ с payload включающим plaintext token

2. worker tasks/run_job.run_job(job_id):
   - SELECT FOR UPDATE jobs SET status='running' WHERE id=... AND status='queued'
   - читает manifest agent_version
   - если runtime.llm:
       llm_config = LlmRuntimeConfig(
           ephemeral_token=payload['ephemeral_token'],
           agents_network_name='portal-agents-net',
           proxy_base_url='http://portal-api:8000/llm/v1',
       )
   - run_agent_container(..., llm_config=llm_config)

3. docker_runner.run_agent_container:
   - if llm_config:
       network='portal-agents-net'
       env += {OPENROUTER_API_KEY, OPENROUTER_BASE_URL}
     else:
       network_mode='none'
   - container.run(...)

4. Агент внутри контейнера (произвольный код студента):
   from openai import OpenAI
   client = OpenAI(
       api_key=os.environ['OPENROUTER_API_KEY'],
       base_url=os.environ['OPENROUTER_BASE_URL'],
   )
   resp = client.chat.completions.create(model='deepseek/deepseek-chat', messages=[...])

5. portal-api роутер POST /llm/v1/chat/completions:
   a. EphemeralTokenAuth: SELECT * FROM llm_ephemeral_tokens
        WHERE token_hash=sha256(bearer) AND revoked_at IS NULL AND expires_at > now()
      → resolves user_id, agent_version_id, agent_id, job_id (или 401 invalid_ephemeral_token)
   b. validate request.model ∈ agent_version.manifest.runtime.llm.models (или 403 model_not_in_whitelist)
   c. pre-flight check в короткой транзакции с FOR UPDATE на user_quotas row:
        - lazy month reset если period_starts_at + 1 month <= now()
        - estimated_cost = (estimated_prompt_tokens × pricing.prompt) + (estimated_completion_tokens × pricing.completion)
        - budget = min(remaining_monthly, remaining_per_job)
        - если estimated > budget → 402 quota_exhausted | per_job_cap_exceeded
        - commit (lock освобождается до forward, чтобы streaming не держал lock на час)
   d. forward к https://openrouter.ai/api/v1/chat/completions с реальным OPENROUTER_API_KEY из .env
      - если stream=True: переопределяем stream_options.include_usage=True
      - проксируем chunks как text/event-stream (StreamingResponse)
   e. по завершении (или после non-stream response) — отдельная короткая транзакция:
      - real_cost = calc_cost(model, response.usage.prompt_tokens, response.usage.completion_tokens)
      - SELECT FOR UPDATE user_quotas WHERE user_id = ...
      - INSERT INTO llm_usage_logs (...)
      - UPDATE user_quotas SET period_used_usd += real_cost
      - UPDATE jobs SET cost_usd_total += real_cost
      - commit
   f. вернуть response клиенту (для non-stream — JSON, для stream — закрыть SSE)

6. tasks/run_job.run_job (продолжение):
   - после run_agent_container (любой исход):
       UPDATE llm_ephemeral_tokens SET revoked_at = now() WHERE job_id = ...
   - финализация job status (succeeded/failed/cancelled/timed_out)
```

### Безопасность

- `internal: true` на agents-network = no NAT. Агент **физически не выйдет в интернет** — TCP до openrouter.ai не установится. Если агент захардкодил другой OpenRouter-ключ — соединение DROP.
- ephemeral-token валиден только для своего job_id и только пока job не в терминальном статусе.
- В БД хранится только sha256, plaintext не восстановим из утечки БД.
- Глобальный whitelist моделей (`LLM_ALLOWED_MODELS` в `.env` portal-api): админ контролирует общий набор разрешённых моделей. Если в whitelist'е нет дорогих моделей — ни один агент их не запросит даже теоретически.
- Per-agent whitelist (`manifest.runtime.llm.models`) проверяется при build (manifest_loader.py) — невалидный manifest не публикуется как `agent_version`.

### Изоляция между агентами

В `portal-agents-net` могут одновременно работать несколько agent-контейнеров. Они видят друг друга по сетевому namespace (Docker не блокирует ICC по умолчанию), но:
- Агенты не открывают портов (только outbound к проксе) — нет surface для соседа.
- Каждый получает свой ephemeral-token — соседский ключ ему не поможет (он привязан к чужому job_id).
- Per-job изоляция файлов уже обеспечена 1.2.3 (отдельные input/output volumes).

Если в будущем потребуется hard isolation (network-per-job) — добавим в 1.2.5+.

## 4. БД-схема

### `user_quotas`

```sql
CREATE TABLE user_quotas (
    user_id            UUID PRIMARY KEY REFERENCES users(id) ON DELETE CASCADE,
    monthly_limit_usd  NUMERIC(10, 4) NOT NULL DEFAULT 5.0000,
    period_used_usd    NUMERIC(10, 4) NOT NULL DEFAULT 0.0000,
    period_starts_at   TIMESTAMP WITH TIME ZONE NOT NULL,
    per_job_cap_usd    NUMERIC(10, 4) NOT NULL DEFAULT 0.5000,
    created_at         TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT now(),
    updated_at         TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT now()
);
```

- Создаётся в `services/auth.register_user` (insert после создания user).
- Backfill: миграция `0004_llm_proxy` инсертит row для каждого существующего юзера.
- `admin` юзеры: `monthly_limit_usd = 999999.9999` (фактически безлимит, без NULL-ветвлений в коде).
- `period_starts_at` хранится в UTC; reset вычисляется `floor_to_month_start_msk(now())` и конвертируется в UTC.
- Lazy reset: при первом запросе после смены месяца `services/llm_quota.check_and_reset_period` срабатывает атомарно с pre-flight check (один UPDATE ... WHERE id=... FOR UPDATE).

### `llm_ephemeral_tokens`

```sql
CREATE TABLE llm_ephemeral_tokens (
    token_hash        VARCHAR(64) PRIMARY KEY,
    job_id            UUID NOT NULL REFERENCES jobs(id) ON DELETE CASCADE,
    user_id           UUID NOT NULL REFERENCES users(id),
    agent_version_id  UUID NOT NULL REFERENCES agent_versions(id),
    created_at        TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT now(),
    expires_at        TIMESTAMP WITH TIME ZONE NOT NULL,
    revoked_at        TIMESTAMP WITH TIME ZONE
);
CREATE INDEX llm_ephemeral_tokens_job_id_idx ON llm_ephemeral_tokens(job_id);
CREATE INDEX llm_ephemeral_tokens_expires_at_idx ON llm_ephemeral_tokens(expires_at);
```

- Plaintext-формат: `por-job-<uuid4_hex>` (33 символа). UUID4 → 122 бита энтропии, достаточно.
- `expires_at = created_at + manifest.limits.max_runtime_minutes minutes + 5 minute buffer`.
- Buffer покрывает финальный chunk OpenRouter после soft-killa контейнера.
- Resolve в прокси:
  ```sql
  SELECT user_id, agent_version_id, job_id
  FROM llm_ephemeral_tokens
  WHERE token_hash = $1 AND revoked_at IS NULL AND expires_at > now()
  ```
- Очистка протухших: вне scope MVP. Можно cron-job'ом раз в день удалять `WHERE expires_at < now() - INTERVAL '7 days'`.

### `llm_usage_logs`

```sql
CREATE TABLE llm_usage_logs (
    id                     UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    job_id                 UUID NOT NULL REFERENCES jobs(id) ON DELETE CASCADE,
    user_id                UUID NOT NULL REFERENCES users(id),
    agent_id               UUID NOT NULL REFERENCES agents(id),
    agent_version_id       UUID NOT NULL REFERENCES agent_versions(id),
    model                  VARCHAR(128) NOT NULL,
    prompt_tokens          INTEGER NOT NULL,
    completion_tokens      INTEGER NOT NULL,
    total_tokens           INTEGER NOT NULL,
    cost_usd               NUMERIC(10, 6) NOT NULL,
    latency_ms             INTEGER NOT NULL,
    status                 VARCHAR(32) NOT NULL,
    openrouter_request_id  VARCHAR(128),
    created_at             TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT now()
);
CREATE INDEX llm_usage_logs_user_created_idx ON llm_usage_logs(user_id, created_at DESC);
CREATE INDEX llm_usage_logs_job_idx          ON llm_usage_logs(job_id);
CREATE INDEX llm_usage_logs_agent_created_idx ON llm_usage_logs(agent_id, created_at DESC);
```

- Пишется на каждый LLM-вызов, даже на ошибку (401/402/403/502 — со `status` и `cost_usd=0`/`tokens=0` где применимо).
- `status` enum: `success`, `quota_exhausted`, `per_job_cap_exceeded`, `model_not_in_whitelist`, `invalid_ephemeral_token`, `openrouter_upstream_error`, `not_implemented`.
- `agent_id` хранится в дополнение к `agent_version_id` (избыточность для индекса `(agent_id, created_at)`, нужна для будущих per-agent aggregations без join'а).
- Retention: на MVP не чистим. При росте — partitioning по `created_at` (месяц).

### `jobs` — расширение

```sql
ALTER TABLE jobs ADD COLUMN cost_usd_total NUMERIC(10, 6) NOT NULL DEFAULT 0;
```

- Накапливается транзакционно вместе с UPDATE `user_quotas`.
- Используется для проверки `per_job_cap_usd` и для отображения юзеру.

### Миграция `0004_llm_proxy`

Одна Alembic-миграция:
1. `CREATE TABLE user_quotas`.
2. `INSERT INTO user_quotas (user_id, period_starts_at) SELECT id, ... FROM users` — backfill.
3. `CREATE TABLE llm_ephemeral_tokens`.
4. `CREATE TABLE llm_usage_logs`.
5. `ALTER TABLE jobs ADD COLUMN cost_usd_total ...`.
6. `UPDATE user_quotas SET monthly_limit_usd=999999.9999 WHERE user_id IN (SELECT id FROM users WHERE role='admin')`.

Downgrade: drop таблицы + drop column.

## 5. API контракт

### Роутер `/llm/v1/*` (OpenAI-совместимый)

Auth: `Authorization: Bearer <ephemeral_token>`. Cookie-сессия игнорируется.

#### `POST /llm/v1/chat/completions`

- Request body: OpenAI Chat Completions API спецификация (с обязательным `model`, `messages`).
- Если в request body `model` не входит в `agent_version.manifest.runtime.llm.models` → `403 {error: {code: "model_not_in_whitelist", message: "..."}}`.
- Pre-flight estimate cost:
  - `estimated_prompt_tokens = len(json.dumps(messages, ensure_ascii=False)) // 3`
  - `estimated_completion_tokens = request.max_tokens or model_pricing.context_length`
  - `estimated_cost = estimated_prompt_tokens × pricing.prompt + estimated_completion_tokens × pricing.completion`
- Если `estimated_cost > min(remaining_monthly, remaining_per_job)` → `402` с конкретным `code` (`quota_exhausted` или `per_job_cap_exceeded`) и payload включающим `current_used`, `limit`, `estimated_cost`.
- Forward к `https://openrouter.ai/api/v1/chat/completions` с `Authorization: Bearer $OPENROUTER_API_KEY` (из portal-api `.env`).
- Если `stream=True`:
  - Override request body: `stream_options.include_usage = True`.
  - Возвращаем `StreamingResponse(media_type="text/event-stream")`, проксируем chunks 1-в-1.
  - Парсим финальный chunk на `usage` для post-flight записи.
  - Если stream оборвался до usage chunk: `completion_tokens = len(streamed_content) // 3`, log warn.
- Если `stream=False`:
  - Возвращаем JSON 1-в-1 от OpenRouter.
  - `usage` берём из response.
- Post-flight (одна транзакция под FOR UPDATE на user_quotas):
  - INSERT `llm_usage_logs`.
  - UPDATE `user_quotas.period_used_usd += real_cost`.
  - UPDATE `jobs.cost_usd_total += real_cost`.
- Если OpenRouter вернул 4xx — пробрасываем status и body клиенту (агент видит реальную ошибку модели, например invalid messages).
- Если OpenRouter вернул 5xx — `502 {error: {code: "openrouter_upstream_error", message, upstream_status}}`.
- Если OpenRouter timeout (30с по умолчанию) — `504 {error: {code: "openrouter_timeout"}}`.

#### `POST /llm/v1/completions`

Аналогично, legacy completions API.

#### `GET /llm/v1/models`

Возвращает только модели из `agent_version.manifest.runtime.llm.models`, формат OpenAI-совместимый:
```json
{
  "object": "list",
  "data": [
    {"id": "deepseek/deepseek-chat", "object": "model", "owned_by": "openrouter"},
    ...
  ]
}
```

Не делает запрос к OpenRouter (тянем из локального кеша pricing для метаданных). Это не публичный listing — это «что доступно текущему агенту».

#### Прочие OpenAI-эндпоинты

`/embeddings`, `/images/generations`, `/audio/*`, `/files/*`, `/fine_tuning/*`, etc. → `501 {error: {code: "not_implemented", message: "Endpoint X not supported by portal proxy on MVP"}}`.

### Расширения существующих эндпоинтов

#### `GET /api/me`

Дополнительно возвращает:
```json
{
  ...,
  "quota": {
    "monthly_limit_usd": "5.0000",
    "period_used_usd": "1.2345",
    "period_starts_at": "2026-05-01T00:00:00+03:00",
    "per_job_cap_usd": "0.5000"
  }
}
```

#### `GET /api/me/usage?from=ISO&to=ISO&cursor=&limit=50`

Возвращает страницу из `llm_usage_logs` для текущего юзера:
```json
{
  "items": [
    {
      "ts": "2026-05-03T15:23:01+03:00",
      "agent_slug": "proverka",
      "model": "deepseek/deepseek-chat",
      "prompt_tokens": 1234,
      "completion_tokens": 567,
      "cost_usd": "0.000345",
      "status": "success",
      "job_id": "..."
    },
    ...
  ],
  "next_cursor": "..."
}
```

#### `GET /api/admin/users/{id}` (admin only)

Возвращает user + поле `quota` (как в `/api/me`).

#### `PATCH /api/admin/users/{id}/quota` (admin only)

Body:
```json
{"monthly_limit_usd": "10.0000", "per_job_cap_usd": "1.0000"}
```

Оба поля опциональные. Возвращает обновлённую `quota`. Не сбрасывает `period_used_usd`.

#### `POST /api/admin/users/{id}/quota/reset` (admin only)

Body: пустое. Сбрасывает `period_used_usd = 0` (но не меняет `period_starts_at`). Возвращает обновлённую `quota`.

#### `GET /api/admin/usage?user_id=&agent_id=&from=&to=` (admin only)

Aggregation на `llm_usage_logs`:
```json
{
  "total_cost_usd": "12.3456",
  "total_requests": 1234,
  "by_user": [{"user_id": "...", "email": "...", "cost_usd": "...", "requests": ...}, ...],
  "by_agent": [{"agent_id": "...", "slug": "...", "cost_usd": "...", "requests": ...}, ...],
  "by_model": [{"model": "...", "cost_usd": "...", "requests": ...}, ...]
```

## 7. Сервисы portal-api

### `services/llm_pricing.py`

```python
@dataclass
class ModelPricing:
    model: str
    prompt_per_token: Decimal
    completion_per_token: Decimal
    context_length: int

class PricingCache:
    def __init__(self): self._data: dict[str, ModelPricing] = {}
    async def get(self, model: str) -> ModelPricing: ...
    async def refresh(self) -> None: ...  # GET https://openrouter.ai/api/v1/models
```

- При старте portal-api FastAPI lifespan: запускает `asyncio.create_task(periodic_refresh(interval=6h))`.
- `get(model)` — если cache miss → forced refresh, если всё равно miss → worst-case fallback (`Decimal("0.00010")` per token).
- При refresh fail (network / OpenRouter 5xx) — оставляет старый cache, log warn.
- НЕ persistится в БД.

### `services/llm_quota.py`

```python
async def preflight(
    session, *, user_id, job_id, estimated_cost: Decimal,
) -> None | QuotaError:
    # SELECT ... FROM user_quotas WHERE user_id = ... FOR UPDATE
    # lazy reset если истёк период
    # check estimated_cost <= min(remaining_monthly, remaining_per_job)
    # raise QuotaExhausted / PerJobCapExceeded если не хватает
    # (commit вне функции)

async def postflight(
    session, *, user_id, job_id, real_cost: Decimal,
) -> None:
    # UPDATE user_quotas SET period_used_usd += real_cost WHERE user_id = ... (под FOR UPDATE того же rowа)
    # UPDATE jobs SET cost_usd_total += real_cost WHERE id = ...
    # (commit вне функции)
```

Pre-flight и post-flight — две **отдельные короткие транзакции**, каждая под `SELECT FOR UPDATE` на `user_quotas` row. Lock не держится на длину OpenRouter-запроса (иначе streaming на час заблокировал бы все остальные вызовы юзера). Конкретно:

- **Pre-flight:** open transaction → FOR UPDATE на user_quotas → lazy reset → check `estimated_cost <= budget` → если не проходит, raise → **commit**. INSERT в `llm_usage_logs` НЕ делаем (нет промежуточного `pending` статуса в схеме).
- **Forward к OpenRouter** (без lock'а на БД).
- **Post-flight:** open transaction → FOR UPDATE на user_quotas → INSERT `llm_usage_logs` (status=`success` или error-status с tokens=0) → UPDATE `user_quotas.period_used_usd += real_cost` → UPDATE `jobs.cost_usd_total += real_cost` → **commit**.

**Известная race:** между pre-flight check и post-flight write нет lock'а. Параллельные запросы того же юзера могут оба пройти pre-flight, оба сделать вызов, оба записать → итоговый расход немного больше cap'а. На MVP принимаем (студенческий агент редко шлёт >5 параллельных). Если станет проблемой — переходим на reservation-pattern (variant 2 в roadmap follow-ups).

При оборванном connection (клиент disconnect, OpenRouter timeout): post-flight всё равно срабатывает с тем, что успели насчитать.

## 8. Pricing / cost calculation

Источник: `GET https://openrouter.ai/api/v1/models` (открыто, без auth).

Response пример (упрощённо):
```json
{
  "data": [
    {
      "id": "deepseek/deepseek-chat",
      "context_length": 64000,
      "pricing": {"prompt": "0.00000014", "completion": "0.00000028"}
    },
    ...
  ]
}
```

Цены — **USD per token** (не per 1M tokens). Хранятся как `Decimal` для precision.

```python
def calc_cost(model: str, prompt_tokens: int, completion_tokens: int) -> Decimal:
    pricing = pricing_cache.get(model)
    return (
        Decimal(prompt_tokens) * pricing.prompt_per_token
        + Decimal(completion_tokens) * pricing.completion_per_token
    )
```

OpenRouter возвращает `usage: {prompt_tokens, completion_tokens}` в каждом ответе — берём оттуда, сами не tokenize'ем.

## 9. Streaming details

OpenRouter SSE формат:
```
data: {"choices": [{"delta": {"content": "..."}}], ...}\n\n
data: {"choices": [{"delta": {"content": "..."}}], ...}\n\n
...
data: {"choices": [], "usage": {"prompt_tokens": ..., "completion_tokens": ...}}\n\n
data: [DONE]\n\n
```

Прокси:
1. Перед forward override `request.stream_options.include_usage = True`.
2. Открывает `httpx.AsyncClient.stream("POST", openrouter_url, json=body, headers={...})`.
3. Async generator: для каждого chunk — yield to client, parse если это JSON-chunk с `usage` → запоминаем.
4. После последнего chunk → post-flight write с usage.
5. Если client disconnect: cancel httpx stream, post-flight с тем что есть.

`StreamingResponse(generator(), media_type="text/event-stream")`.

## 13. Конкретные риски и митигации

| Риск | Митигация |
|---|---|
| Pre-flight worst-case оценка отрезает безобидные запросы (юзер не указал max_tokens, модель имеет 200k context) | Документируем «всегда указывайте max_tokens». Если станет проблемой — переходим на tiktoken для известных моделей. |
| OpenRouter недоступен → весь портал не может запускать LLM-агенты | По умолчанию все запросы 502. Юзер видит ошибку, может попробовать позже. Не критично, не SLO. |
| Race на user_quota при параллельных запросах от агента | FOR UPDATE на user_quotas row в pre-flight + post-flight транзакциях. |
| Ephemeral token утёк в логи | Логируем только hash. plaintext только в env переменной контейнера. |
| Agent попытается выйти в интернет напрямую | `internal: true` Docker network гарантирует no-NAT. |
| Pricing cache не обновился — устарел | Background refresh каждые 6ч + forced refresh при cache miss. Worst-case fallback при полном fail. |
| Stream оборвался до usage chunk → недосчитали стоимость | Оцениваем completion_tokens из streamed-content size. Логируем warn. Юзер недоплатит мизерные суммы. |
| Backfill `user_quotas` упал на проде с большим числом юзеров | Сейчас юзеров мало (dev). Миграция простая INSERT-SELECT. На проде с 1000+ — батчить, но не сейчас. |
