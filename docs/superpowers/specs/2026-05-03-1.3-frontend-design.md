## 1. Цель и контекст

После 1.2.4 backend полностью функционален: auth, реестр агентов с git+docker build, запуск job'ов с live-стримом по WebSocket, LLM-прокси с квотами, admin endpoints для всего. Но взаимодействие — только через `curl` и `psql`. **Цель 1.3 — построить Next.js frontend, который позволит преподам НУГ пользоваться платформой без CLI и Python**, а админу — управлять реестром и квотами через UI.

Целевая аудитория:
- **Преподаватели НУГ (primary)** — запускают агентов на работах студентов, скачивают результаты. 35-55 лет, не разработчики. Должны почувствовать «солидный продукт, а не игрушка».
- **Студенты НУГ (secondary)** — пишут агенты по контракту, тестируют их через портал. 18-25 лет, разработчики.
- **Админ (один-два человека)** — публикует agent_versions, управляет квотами, выдаёт invites.

После merge 1.3:
- Любой preподаватель НУГ может зайти на `portal.<домен>` → войти → запустить агента на своих файлах → получить результат с live-прогрессом.
- Студент может тестировать свой агент после публикации.
- Админ может выдавать invites, поднимать квоты, публиковать новые agent_versions без обращения к разработчику.

**После merge 1.3 платформа технически готова к запуску в НУГ.**

## 2. Scope

### Входит ✅

- Next.js 15 (App Router) приложение в `apps/portal-frontend/`
- 12 страниц: 3 public (landing/login/register-by-invite) + 5 authorised (agents catalog, agent detail, jobs list, job detail, profile) + 4 admin (users, agents, tabs, usage)
- Дизайн-система в стиле Anthropic-школы: paper-фон, сериф для заголовков, sans для UI, brick-red accent (точные hex/шрифты — design-consultation после плана)
- Полная shadcn/ui кастомизация (форк в наш репо, override через CSS vars)
- Cookie-auth flow + middleware + refresh
- Server Components для данных + TanStack Query для мутаций
- WebSocket для live job progress (с reconnect + resync через `?since=N`)
- Multipart file upload через FormData
- Динамическая форма по `manifest.inputs/files`
- Admin tables: pagination, sort, filter, drawer pattern
- Empty / error / loading states везде
- `lib/api/errors.ts` с маппингом backend `error.code` → user-friendly RU-string
- 5-7 Playwright e2e сценариев (auth, run job, live stream, quota exhausted, admin: create invite, publish agent_version, usage)
- 5 Vitest unit-файлов (agent-form, file-upload, job-stream, api errors, auth middleware)
- TypeScript strict, ESLint, Prettier
- Dockerfile + интеграция в существующий `docker-compose.yml` как сервис `frontend`
- Только русский UI (без i18n инфраструктуры)
- Desktop-first, responsive не ломаем (down to 375px)

- Mobile-first оптимизации (touch gestures, swipe-навигация, etc.)
- Dark mode (можно добавить позже через CSS vars)
- Storybook (overkill для текущего размера команды)
- Snapshot-тесты компонентов (хрупкие)
- Visual regression тесты (позже через `design-review` skill)
- i18n RU/EN (только RU)
- Time-series графики в `/admin/usage` (только totals по date range)
- WebHooks / уведомления о завершении job'ов (отложено)
- Drag-to-reorder на табах (если успеваем — добавим, иначе CRUD без)
- Server-side rendering optimisations (RSC по дефолту достаточно)
- CMS / редактируемые тексты на landing (захардкоженный markdown)

### Поток данных

```
Browser
  │
  ├── HTTP request → Next.js middleware (auth refresh) → Route handler
  │   └── Server Component → apiServer (cookie из request) → fetch backend
  │       └── Render HTML с данными
  │
  └── Client interaction (form submit, button click)
      └── TanStack Query mutation → apiClient (browser cookie auto) → POST/PATCH
          └── invalidate queries → refetch → UI update

WebSocket
  Browser → WS handshake (cookie auto) → /api/jobs/{id}/stream?since=N
    └── ws.onmessage → dispatch events → JobStream component re-render
```

### Auth flow подробно

1. Незалогиненный заходит на `/agents` → middleware: нет cookie → redirect на `/login?next=/agents`.
2. Login → POST `/api/auth/login` → Set-Cookie: `access_token` + `refresh_token` → redirect на `next` или `/agents`.
3. Через 24ч access_token истёк → middleware фиксит на следующем request:
   - Server-side fetch к `/api/auth/refresh` с `refresh_token` cookie
   - Получаем новые Set-Cookie headers — пробрасываем в response
   - Продолжаем оригинальный request
4. Если refresh не сработал (refresh_token истёк) → redirect `/login?next=<original>`.
5. Logout → POST `/api/auth/logout` → backend очищает cookies → redirect `/`.

## 4. Дизайн-система

**Финальные значения hex/шрифтов будут проработаны через `design-consultation` skill после написания плана.** Здесь — направление и границы.

### Палитра

| Слой | Назначение | Направление |
|---|---|---|
| `--bg-primary` | Основной фон | Тёплый paper, не белый. `#F4EFE6` или `#FAF7F2` (точно — design-consultation) |
| `--bg-secondary` | Cards, inputs | Чуть темнее warm grey, например `#EDEAE2` |
| `--text-primary` | Заголовки, body | Charcoal `#2B2B2B` |
| `--text-secondary` | Метаданные, hint | Warm grey `#6B6560` |
| `--accent` | Primary CTA, links | **brick red `#8B2E1F`** (рекомендация). Альтернативы: deep teal `#0D5856`, olive `#5C5D33` |
| `--success` | Job succeeded | Сдержанный moss green `#5C7C40` |
| `--warning` | Quota close, warnings | Терракота `#C77B3F` |
| `--error` | Job failed, 402 | Глубокий багровый `#8B2E1F` (от accent) |
| `--info` | Running job, info | Pewter blue `#4A5568` |
| `--border` | Тонкие линии | Warm grey `#D9D4CC` |

**Не используем:** яркие неоновые цвета, чёрный (только charcoal), gradient, glassy эффекты, dark mode (на MVP).

### Типографика

- **Headings (H1-H4):** open-source serif с характером, кириллица обязательна. Кандидаты:
  - **Newsreader** (Google Fonts, RU support)
  - **Instrument Serif** (Google Fonts, RU support)
  - **EB Garamond** (Google Fonts, RU support)
- **UI / body:** geometric sans без китча.
  - **Geist Sans** (open source, Vercel)
  - **Inter** (open source, нейтральный)
- **Mono для метаданных, чисел, кода:** **Geist Mono** или **JetBrains Mono**.

Modular scale (1.25x): `12 / 14 / 16 / 18 / 22 / 28 / 36 / 44 / 56`.

### Spacing

4px baseline: `4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96`.

Spacious, не tight. Большие padding'и в cards (`24px+`), заметные зазоры между секциями (`48-96px`).

### Компоненты

shadcn/ui как baseline, форк в `components/ui/`. Кастомизация:
- `Button` — добавить наш accent color, тонкие edges (border-radius 4px не 8px), сериф-mono mix для labels (?)
- `Card` — paper фон, тонкий border-1 + минимальная тень
- `Input/Textarea` — paper фон, border-bottom выраженный, не box
- `Drawer` — slide from right, paper фон
- `Dialog` — overlay charcoal с opacity 60%, paper modal
- `Badge` — для статусов job'ов (queued/running/succeeded/failed/cancelled)
- `Toast` — sonner библиотека, paper стиль

### Иконки

Lucide React (тонкий outline). Логотип портала — отдельный custom mark (геометрический, разработать в design-consultation).

### Motion

Сдержанный. Easing: `ease-out` для enter, `ease-in` для exit. Длительности 150-300ms.

Без Framer Motion на каждый элемент. Только:
- Drawer slide (300ms)
- Modal fade (200ms)
- Page transitions (опционально, через next/transitions)
- Toast appear/disappear (250ms)

## 5. Информационная архитектура

### Public (не-авторизованные)

#### `/` — Landing
- Hero: «Портал НУГ "Цифровые технологии в математическом образовании" → AI-агенты для рутинных задач НУГ»
- 3-4 примера агентов (карточки): «Эксперт конкурсных работ», «Поиск научных статей», «Проверка работ» — статичные, не data-driven (захардкожены в landing)
- CTA `Войти`. Текст под CTA: «Доступ — по приглашению. Попроси админа НУГ.»
- Опционально: footer со ссылкой на GitHub репо (если public).
- Один экран, не лонгрид.

#### `/login`
- Email + password, кнопка `Войти`.
- Опционально: «Забыл пароль?» (показывает alert «Свяжись с админом» — full-flow на MVP не делаем).
- На сабмит: POST `/api/auth/login`. Success → redirect `next || /agents`. Error → toast.

#### `/register?invite=<token>`
- Email auto-filled из decoded invite token (через preflight GET `/api/auth/invite-info?token=...` или хардкод в URL).
- Поля: пароль + confirm password, согласие с правилами (checkbox).
- На сабмит: POST `/api/auth/register` с `token` + `password`. Success → auto-login → redirect `/agents`.
- Если invite не валиден → красивая ошибка: «Приглашение истекло или уже использовано. Свяжись с админом.»

### Authorised

#### `/agents` — Каталог агентов
- Layout: sidebar с табами (категории) + main content с карточками агентов.
- На mobile (<768px): sidebar схлопывается в горизонтальные tabs.
- Каждая карточка: icon (emoji или PNG из manifest) + name + short_description + кнопка `Открыть`.
- Empty state если нет агентов: иллюстрация + текст «Пока нет ни одного агента. Спроси админа.»

#### `/agents/{slug}` — Страница агента
- Header: large H1 с name, badge tab, иконка/emoji.
- Body: длинный `about` (markdown render через react-markdown).
- Right column (или ниже на mobile): `<AgentForm />` — динамическая форма по `manifest.inputs` + `manifest.files`. Кнопка `Запустить` внизу.
- На сабмит: redirect на `/jobs/{id}`.
- Если агент disabled: form скрыта, banner «Агент временно недоступен».

#### `/jobs` — Мои job'ы
- Таблица (TanStack Table): агент (slug), статус (badge), создано, длительность, cost (USD если LLM), actions.
- Фильтры (URL state): status (multi-select), agent_slug, period (last 7d / 30d / custom).
- Cursor pagination.
- Click row → `/jobs/{id}`.
- Empty state: «Нет задач. [Запустить агента →]»

#### `/jobs/{id}` — Детали job
- Header: H1 с agent name, status badge, дата создания, кнопки `Отменить` (если running) / `Перезапустить` (если failed/cancelled).
- Two-column layout (на mobile — stacked):
  - **Слева (live):** `<JobStream />` — events feed с timestamps, progress bar поверху, current status.
  - **Справа (метаданные):** input params (collapsible JSON), список output artifacts (когда status=succeeded) с кнопками `Скачать`.
- На терминальном статусе — скрываем cancel button, показываем `Перезапустить`.
- Если job чужой — 403, redirect `/jobs`.

#### `/me` — Профиль
- Tabs: `Профиль` / `Квота` / `История LLM`.
- **Профиль:** read-only поля (email, role, created_at), кнопка `Сменить пароль` (открывает modal: current + new + confirm).
- **Квота:** прогресс-бар `period_used_usd / monthly_limit_usd` с подписью «осталось $X.XX до 1-го числа». per_job_cap отдельной строкой. Плюс пояснение «Что такое квота».
- **История LLM:** таблица из `/api/me/usage`: ts / agent / model / tokens / cost / status. Cursor pagination. Date range filter.

### Admin

#### `/admin/users` — Список юзеров
- Таблица: email / role / created / `period_used / monthly_limit USD` / actions.
- Inline edit для квоты (клик на cell → input → blur → PATCH).

#### `/admin/agents` — Список агентов

**Submit:** multipart/form-data, `params` как JSON, файлы по `files[id]` keys. POST через `apiClient`. На 402 quota_exhausted → toast + не очищать форму.

**Сложности:** dynamic Zod schema, multipart с прогрессом, валидация max_total_size_mb.

### 6.2 `<JobStream jobId={...} />`

**Файл:** `components/job-stream/JobStream.tsx`

Live прогресс job через WebSocket с reconnect.

**Поток:**
1. Mount → fetch initial state: `GET /api/jobs/{id}` + `GET /api/jobs/{id}/events?since=0`.
2. Подключение WebSocket: `wss://.../api/jobs/{id}/stream?since=N`.
3. На message: parse JSON event → dispatch к local state.
4. Heartbeat: ping каждые 30s. Если no pong 60s → reconnect.
5. На reconnect: `?since=lastSeq`, чтобы не пропустить events.
6. На терминальном event (result / failed / cancelled / timed_out) → close WS, fetch artifacts.
7. На unmount → close WS.

**UI:**
- Progress bar (если приходят `progress` events с float 0-1).
- Status badge сверху (queued/running/succeeded/failed/cancelled).
- Vertical feed events с timestamps, разные стили для типов:
  - `started` — серая «hello»
  - `progress` — обновляет bar, не feed
  - `item_done` — зелёная check
  - `log` — серая моноширинная строка
  - `result` — большая зелёная заметка
  - `failed` / `error` — красная

## 7. Auth + state management

### Cookie auth

Backend выдаёт два HttpOnly cookies (`access_token` 24ч, `refresh_token` 30 дней). Никакого client-side хранения токенов.

### Next.js middleware

Файл `middleware.ts`:
- Если route в `(public)/` или статика — пропустить.
- Если есть валидный access_token cookie — пропустить (RSC сам провалит fetch если 401).
- Если access_token отсутствует/expired, но есть refresh_token:
  - Server-side fetch к `/api/auth/refresh` с cookie.
  - Получаем новые Set-Cookie из response.
  - Создаём NextResponse с новыми cookies, продолжаем.
- Если refresh не сработал — redirect на `/login?next=<original>`.

### RSC server fetch (`lib/api/server.ts`)

```ts
import { cookies } from 'next/headers';

export async function apiServer<T>(
  path: string, init?: RequestInit,
): Promise<T> {
  const cookieHeader = cookies().toString();
  const res = await fetch(`${process.env.API_BASE_URL}${path}`, {
    ...init,
    headers: { ...init?.headers, Cookie: cookieHeader },
    cache: 'no-store',
  });
  if (!res.ok) throw new ApiError(res.status, await res.json());
  return res.json();
}
```

### Client mutations (`lib/api/client.ts`)

Через TanStack Query. Default config: `staleTime: 30s`, `refetchOnWindowFocus: false`, `retry: 1`.

```tsx
'use client';
const mutation = useMutation({
  mutationFn: (data) => fetch('/api/agents/proverka/jobs', {
    method: 'POST', body: data, credentials: 'include',
  }).then(r => r.ok ? r.json() : Promise.reject(r)),
  onSuccess: (job) => router.push(`/jobs/${job.id}`),
  onError: (err) => toast.error(mapApiError(err)),
});
```

### Error handling (`lib/api/errors.ts`)

```ts
const MESSAGES_RU: Record<string, string> = {
  invalid_credentials: 'Неверный email или пароль.',
  invite_expired: 'Приглашение истекло или уже использовано.',
  invite_invalid: 'Ссылка-приглашение неверна.',
  quota_exhausted: 'Месячная квота исчерпана. Обновится 1-го числа.',
  per_job_cap_exceeded: 'На этой задаче исчерпан лимит стоимости.',
  invalid_ephemeral_token: 'Сессия LLM истекла. Перезапусти задачу.',
  model_not_in_whitelist: 'Эта модель не разрешена для агента.',
  not_implemented: 'Эта функция пока не поддерживается.',
  openrouter_upstream_error: 'OpenRouter временно недоступен.',
  openrouter_timeout: 'OpenRouter не ответил вовремя.',
  job_not_found: 'Задача не найдена.',
  agent_not_ready: 'Агент ещё собирается. Попробуй через минуту.',
  // ...
};

export function mapApiError(err: ApiError): string {
  const code = err.body?.error?.code;

```

### State management — что НЕ делаем

- Никакого Redux / Zustand / Jotai. Server state — TanStack Query, UI state — useState.
- URL — single source of truth для drawer state, filters, date ranges.
- Никакого persisted store (cookies + server).

## 11. Конкретные риски и митигации

| Риск | Митигация |
|---|---|
| Дизайн-система не «зайдёт» преподам — слишком bold | Прогон через `design-consultation` (палитра + шрифты) с 2-3 вариантами для выбора. После имплементации — `design-review` для visual QA. |
| Динамическая форма по manifest падает на нестандартных полях | Тесты для всех типов (`text/textarea/number/checkbox/select/radio/date` + 4 file types). Fallback: если type неизвестен → render `<Input type=text>` с warning в консоли. |
| WebSocket reconnect зацикливается | Exponential backoff (max 30s). Если не reconnect'нулся за 5 min — fallback на polling `/api/jobs/{id}` каждые 5s. |
| Auth refresh вызывает race condition | Мьютекс/lock на refresh — пока один в процессе, остальные ждут результата (через single-flight). |
| Преподы не понимают «invite-flow» | На странице `/login` ниже формы — текст «Доступ только по приглашению. Свяжись с админом НУГ.» На register-странице — крупный bullets «Что значит приглашение, что нужно сделать». |
| Mobile-режим ломается | Не оптимизируем mobile-first, но Playwright e2e тест запускает один сценарий на mobile viewport (`375x812`). Если ломается — фиксим. |
| Backend меняет схему — ручные TS-типы расходятся | Используем OpenAPI typegen (`openapi-typescript`) — auto-generate `lib/api/types.ts` из бэкендового `/openapi.json`. Команда `npm run codegen` регенерит. |
| Динамическая загрузка markdown в `manifest.about` уязвима к XSS | `react-markdown` по дефолту escape'ит. Не включаем `rehype-raw`. Кастомные components только для нужных тегов. |
